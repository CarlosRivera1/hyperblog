Resumen de lo visto hasta esta clase: git config --global user.name “Fulanito de tal”: En git es necesario identificar que usuario hace los commit, por lo tanto antes de realizar nuestro primer commit es necesario identificarnos. Con este comando le señalamos a git quien está realizando los commit al proyecto. git config --global user.email fulanito@correo.co: Con este comando le señalamos a git el correo del usuario que realiza los commit. git commit: Guarda de manera definitiva en git los cambios realizados a los archivos que estaban en etapa de staging. Admite las siguiente instrucciones: git commit -m “Comentario”: añade un comentario a nuestro commit (obligatorio, puesto que requerimos saber que cambios hicimos a nuestro proyecto en este commit) git commit -a: Si no se han añadido los cambios a staging con git add, este comando fusiona git add y git commit. git log: Muestra un historial de los commit que hemos realizado en la rama donde estamos ubicados actualmente. git reset : Destruye todos los cambios que se han realizado a partir del commit que especifiquemos, volviendo al estado en el que estabamos dentro de nuestro proyecto en ese commit. Hay dos maneras de realizarlo: git reset --soft: Volvemos a la versión especificada, pero los cambios en staging aun quedan en ese estado, listos para realizar un git add git reser --hard: Volvemos a la versión especificada y eliminamos del staging los cambios que esten en ese estado. git diff: Muestra los cambios que se han realizado al archivo entre el ultimo git add (los que están en staging) y el archivo que tengo en disco pero que aun no está en staging.
